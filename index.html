Formularelemente und Attribute in HTML5
Erleuterung und Demonstration einiger Formularelemente und deren Attribute in HTML5

Benutzerfreudlichkeit bei Formularen

Semantisch korrekte Auszeichnung als essentieller Bestandteil f. Benutzerfreudlichkeit und Barrierefreiheit.

Die Benutzerfreudlichkeit stellt einen zentralen Aspekt bei der Erstellung (und Gesstaltung) von Formularen dar. Sie sind eine wesentliche Schnittstelle zwischen User und Anbieter einer Website. Dementsprechend tragen Sie einen erheblichen Anteil zum Erfolg einer Website bei.

Beispiele / Demonstrationen

Zum optimalen Nachvollziehen der folgenden Beispiele öffnen Sie die Entwickerwerkzeuge des Browsers, bzw. lassen Sie sich den Quelltext der Seite anzeigen.

Beispiel 1

Vorstellung einiger Formularelemente. Die Beschriftungen (label) sind über das Attribut for jeweils mit dem entsprechenden Eingabefeld verknüpft.
Alternativ können die Eingabefelder auch jeweils innerhalb des Beschriftungselementes plaziert werden. Dann kann auf das for-Attribut verzeichtet werden.
Das Eingabefeld verfügt über die Attribute type="search" und autofocus.

Mit "search" wird das Feld als Suchfeld ausgezeichnet. Die Darstellung kann zwischen verschiedenen Betriebssystemen unterschiedlich sein. Aktuelle Browser können bereits zuvor eingegebene Begriffe als Vorschläge anzeigen.

Durch die Verwendung von autofocus wird das Eingabefeld beim Laden der Seite automatisch fokussiert.

Das Eingabefeld verfügt über die Attribute type="text", placeholder="Ihr Nachname" und required.

Mit "text" wird das Feld als gewöhnliches Textfelde ausgezeichnet.

Mit dem Attribut placeholder und einem passenden Wert kann dem User eine zusätzlich Hilfe beim Ausfüllen des Formulars bereitgestellt werden.

Durch die Verwendung von required wird dieses Feld als obligatisch auszufüllen ausgezeichnet. Kommt der User dem nicht nach wird ein entsprechender Hinweis eingegeblendet.

Das Eingabefeld verfügt über die Attribute type="number" min="0", max="120" und step="3".

Mit type="number" wird das Eingabefeld dahingehend ausgezeichnet, dass dieses ausschließlich Zahlenwerte akzeptiert. Außerdem werden innerhalb des Feldes kleine Pfeile eingeblendet mit denen ein Zahlenwert durch klicken, alternativ zur Eingabe über die Tastatur, eingestellt werden kann.

Mit den Attributen min und max können sinnvolle Minimal- bzw. Maximalwerte definiert werden.

Durch die Verwendung von step wird die Schrittweite definiert, mit der der Wert im Eingabefeld beim klicken auf die Pfeile erhöht, bzw. verringert werden soll.

Das Eingabefeld vom type="date" ermöglicht die Eingabe eines Datums.
Das Eingabefeld vom type="color" ermöglicht die Eingabe/Selektion einer Farbe.
Mit dem Eingabefeld type="range" kann in Form eines Schiebereglers (Slider) ein Wert eingestellt werden.

Wie bei type="number" definieren hier min und max den erlaubten Wertebereich. Ebenso steuert auf hier step die Schrittweite.

Mit dem Attribut value lässt sich ein initaler Wert definieren, der beim laden der Seite eingestellt wird. in diesem Fall ist dies der Wert "5".

Dem Slider schließt sich hier ein weiters Eingabefeld vom type="text" an. Dieses ist jedoch durch Verwendung des Attributes disabled deaktiviert. Das bedeutet, dass in diesem Eingabefeld keine eingaben gemacht werden können.
Stattdessen wird hier der im Slider eingestellte Wert unmittelbar angezeigt.

Das Eingabefeld vom type="email" ermöglicht die Eingabe einer E-Mailadresse.
Es findet eine rudimentäre Validierung statt, indem auf das Vorkommen eines @-Zeichen geprüft wird und bei dessen Fehlen ein entsprechender Hinweis eingeblendet wird, wenn sich der Mauszeiger über dem Eingabefeld befindet.
Das Eingabefeld vom type="url" ermöglicht die Eingabe einer Webadresse.
Auch hier findet eine oberflächliche Validierung der Eingabe statt, indem geprüft wird ob "http://", bzw. "https://" in der Eingabe vorkommt.
Mit dem Eingabefeld type="tel" kann ein Eingabefeld zur Eingabe einer Telefonnummer ausgezeichnet werden.

Auch wenn eine Validierung per JavaScript und/oder Serverseitig erfolgt (was unerlässlich ist!), sollte gleich an dieser Stelle das pattern-Attribut verwendet werden. In diesem Beispiel wird so zumindest erreicht, dass ein Hinweis eingeblendet wird, wenn die Eingabe vom definierten Muster (min. 5, max. 12 Zahlen) abweicht. (Auch hier muss sich der Mauszeiger über dem Eingabefeld befinden, damit der Hinweis angezeigt wird.)

Ein Eingabefeld vom type="text" wird hier mit einem datalist-Element verknüpft. Der User kann so aus einer Liste von Optionen wählen, oder seler eine Eingabe vornehmen. Je nach Anwendungsvall ist dies einem select-Element vorzuziehen.

Die Verknüpfung erfolgt im input-Element unter Verwendung des list-Attributes. Dieses erhält als Wert die ID der zu verwendenden datalist

Innerhalb der datalist befinden sich die Einzelnen option-Elemente, dessen jeweilige value-Attribute, die Auswahlmöglichkeiten sind.

Abweichend von den zuvor vorgestellten Elementen handelt es sich bei select um ein Auswahlelement.

Im Gegansatz zu dem vorherigen Beispiel, bei dem ein type="text"-Eingabeelement mit einem datalist-Element kombiniert wurde, kann der User bei einem select-Element nur zwischen den angebotenen Optionen wählen.

Die Auswahlmöglichkeiten werden wie beim datalist-Element in der Form von option-Elementen ausgezeichnet.
